ROTEADORES: -> Função de conectar redes diferentes!
pc - roteador é preciso cabo crossover?

->>> Lembre-se da regra: - Dois equipamentos iguais estão interconectados? Se sim, eles tem o mesmo tipo de placa, então devo usar o cabo crossover. Se não, faço a pergunta abaixo - Dois equipamentos diferentes estão conectados? Essa conexão representa o que naturalmente o equipamento foi desenvolvido para fazer?

Por exemplo ao interconectar o computador ao hub e o computador ao switch, o computador foi feito para se comunicar com várias máquinas e o hub e switch foram feitos para interconectar diversas máquinas. Dessa forma ao conectarmos os dois, vamos estar explorando o que os dois foram fabricados para fazer naturalmente. Porém o roteador foi feito para interconectar redes, se eu coloco somente um dispositivo, não terei como inserir outros dispositivos para o roteador encaminhar os pacotes e então a totalidade de sua função não está sendo explora. Devemos usar cabo crossover.

CLASSES IP
----
Classe A | intervalo 1º octeto (1 - 126) | máscara de rede (255.0.0.0)
Classe B | intervalo 1º octeto (128 - 191) | máscara de rede (255.255.0.0)
Classe C | intervalo 1º octeto (192 - 223) | máscara de rede (255.255.255.0)
Classe D | intervalo 1º octeto (224 - 239) | máscara de rede (-) -> Classe reservado para uso de multicast, classe reservada.
Classe E | intervalo 1º octeto (240 - 255) | máscara de rede (-) -> classe reservada.

Máscara de rede separa o ip em dois grandes grupos:
1. 255 (rede)
2. 0 (máquinas/hosts)

Dentro das classes A, B, C existem faixas privadas, apenas comunicação local, não pode ser utilizado para comunicação na internet.
Classe A | intervalo privado 10.x.x.x
Classe B | intervalo privado 172.16.x.x - 172.31.x.x
Classe C | 192.168.x.x -> ip privado forcenido pela operadora de internet, porém quando acesso um site na internet o moldem (método NAT) faz a tradução desse meu ip local para um ip que acesso a internet.
NAT -> fazer a tradução do endereço privado para o endereço público


Endereço de rede e endereço de BROADCAST não podem ser usados, mas todos os ips dentro desse intarvalo sim.
IP BROADCAST -> O IP de broadcast é um IP que é utilizado para mandar uma mensagem para todos os dispositivos da rede. Qualquer máquina que estiver na rede pode receber a mensagem. Este tipo de mensagem pode ser útil para casos onde é preciso mandar uma mensagem para uma máquina na rede, mas não se sabe o IP dela. Por exemplo, um cliente DHCP ao entrar uma rede com DHCP utiliza uma mensagem de broadcast para o servidor DHCP fornecer um IP para ele. Mais sobre o DHCP aqui: https://en.wikipedia.org/wiki/Dynamic_Host_Configuration_Protocol.
